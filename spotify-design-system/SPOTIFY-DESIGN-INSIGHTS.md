# Spotify Design System Learning Journey

## What I Learned

### Utility-First CSS with Tailwind
Tailwind utility classes make it fast to build consistent UIs without writing custom CSS.
Custom design tokens in tailwind.config.ts ensure brand colors are reusable everywhere.
The darkMode class strategy allows switching themes by toggling a class on the root element.

### Component-Based Design
Breaking UI into small components (Sidebar, Player, SongCard) makes the codebase maintainable.
Hover states and transitions create polished micro-interactions without JavaScript.

### Theme Management
Dark mode is implemented by toggling the dark class on the document element.
CSS variables and Tailwind extend theme provide consistent design tokens.

### Responsive Design
Tailwind grid system with responsive prefixes handles all screen sizes.
Mobile-first approach ensures the app works on all devices.
